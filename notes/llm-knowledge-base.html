📖 1. NLP 基础概念

NLP vs LLM NLP 基础

NLP（Natural Language Processing，自然语言处理）是一门研究计算机理解和生成人类语言的广泛领域。涵盖情感分析、命名实体识别、机器翻译等众多技术。

LLM（Large Language Model，大语言模型）是 NLP 的一个子集，以其巨大的规模、广泛的训练数据和多任务能力著称。代表：GPT、Llama、Claude 等。

简单说：NLP 是领域，LLM 是当前最强大的工具。

NLP 任务类型 NLP 基础
  • 整句分类 — 情感分析、垃圾邮件检测、语法判断
  • 序列标注 — 词性标注（名词/动词/形容词）、命名实体识别（人名/地点/机构）
  • 文本生成 — 补全、摘要、翻译
  • 问答 — 给定上下文提取答案
  • 跨模态 — 语音转文字、图像描述
LLM 核心特征 NLP 基础
  • 规模巨大 — 数百万到数千亿参数
  • 通用能力 — 无需专门训练即可执行多种任务
  • 上下文学习（In-Context Learning） — 从提示中的示例学习，无需更新参数
  • 涌现能力（Emergent Abilities） — 大模型展现出的、小模型不具备的新能力
LLM 局限性 NLP 基础
  • 幻觉 — 可能自信地生成错误信息
  • 缺乏真正理解 — 纯粹基于统计模式，不能真正"理解"
  • 偏见 — 可能复现训练数据中的偏见
  • 上下文窗口有限 — 虽然正在改善
  • 计算资源需求大 — 训练和推理成本高

  • 🤗 Transformers — 统一的模型接口，支持数千种预训练模型
  • 🤗 Datasets — 高效的数据集加载与处理
  • 🤗 Tokenizers — 高速分词器（BPE、WordPiece、SentencePiece）
  • 🤗 Accelerate — 分布式训练工具

🏗️ 2. Transformer 架构

为什么需要 Transformer？— RNN 的困境 架构

2017 年之前，处理语言的主流模型是 RNN（循环神经网络）。它像一个流水线工人，一次只处理一个词，把"记忆"传给下一步。

致命问题 1：长距离依赖 — 就像"传话游戏"，信息经过多步后逐渐丢失。当需要连接句子开头和结尾的信息时，RNN 已经"忘记"了。

致命问题 2：无法并行计算 — RNN 必须按顺序处理，第二个词必须等第一个词处理完。就像排队过一座单车道的桥。

Transformer 的答案（2017）：用 Attention 机制，每个词可以直接"看到"所有其他词，且所有词同时计算。就像一座立交桥。
Attention Is All You Need 架构

2017 年 Google 团队的里程碑论文，标题说明了核心思想：注意力就是一切。

提出了用纯 Attention 机制替代 RNN 的 Transformer 架构，同时解决了长距离依赖和并行计算两大问题。从此 NLP 领域进入 Transformer 时代。

该论文是 2017 年以来 NLP 领域引用量最高、影响最深的论文之一。

Encoder 和 Decoder 的区别 架构

原始 Transformer 有两个部分：

🔊 Encoder（编码器）

"理解者" — 读懂输入

每个词可以看到所有其他词

双向注意力

适用于：分类、情感分析、语义理解

🎤 Decoder（解码器）

"表达者" — 生成输出

每个词只能看到它前面的词

单向/因果注意力

适用于：文本生成、翻译、对话

Decoder-Only 架构 架构

MiniMind、GPT、LLaMA 等现代模型使用 纯 Decoder 架构，没有 Encoder 部分。所有输入和输出在同一个序列中处理。

通过因果掩码（Causal Mask）确保在训练时，每个 token 只能看到自己及之前的位置，模拟推理时的生成场景。这是目前大模型的主流选择。

Encoder（双向）: "我 喜欢 编程" → "喜欢"可以同时看到"我"和"编程" Decoder（单向/因果）: "我 喜欢 编程" → "喜欢"只能看到"我"，看不到"编程" → "编程"可以看到"我"和"喜欢"

⚙️ 3. 核心组件详解

Embedding（嵌入层） 组件

将离散的 token（词/子词）映射为稠密向量。每个 token 变成一个固定维度的向量（如 MiniMind 中为 768 维）。

类比：就像把"菜单翻译成厨房能理解的语言"，将文字转化为模型能处理的数值形式。

在 MiniMind 中，embedding 层将 vocab_size（6400）× hidden_size（768）的查找表中检索对应向量。

位置编码 — RoPE（旋转位置编码） 组件

Attention 有一个"缺陷"——它是无序的。"小猫追小狗"和"小狗追小猫"对纯 Attention 来说关系一样。因为 Attention 只看"谁和谁相关"，不看"谁在前谁在后"。

RoPE（Rotary Position Embedding）的核心思想是"旋转"——把位置信息编码为向量空间中的旋转角度。两个位置之间的距离越远，旋转角度越大。

位置 0 的词向量 → 旋转 0° 位置 1 的词向量 → 旋转 θ° 位置 2 的词向量 → 旋转 2θ° ...

优势：位置信息自然融入 Q 和 K 的点积运算；可以灵活外推到更长的未见序列。

Self-Attention（自注意力机制） 组件

这是 Transformer 最核心的创新。对于句子中的每个词，计算它应该"关注"其他所有词的程度。

类比：读"小猫坐在垫子上，它很舒服"时，你的大脑会回头看句子，判断"它"指的是"小猫"而不是"垫子"。"小猫"得到了高注意力权重。

输入: "小猫 坐在 垫子 上 它 很 舒服" 当处理"它"时的注意力权重: "小猫" → 0.65 ⭐ "垫子" → 0.12 "坐在" → 0.05 ...
Q、K、V 三剑客 组件

Self-Attention 有三个核心角色，用查百科全书来类比：

🔍 Query（查询）

你的问题 — "我想找什么？"

🏷️ Key（键）

百科标题 — "我这里有什么？"

📄 Value（值）

百科内容 — "如果你来找我，我给你这些信息"

🔄 工作流程

Q·K → 匹配度 → Softmax → 权重 × V

第 1 步: 每个词生成自己的 Q、K、V "小猫" → Q_小猫, K_小猫, V_小猫 "坐在" → Q_坐在, K_坐在, V_坐在 第 2 步: 用 Q 去和所有的 K 做匹配（点积） Q_它 · K_小猫 = 高分 ⭐ Q_它 · K_垫子 = 低分 第 3 步: softmax 转化为注意力权重 [高分, 低分, ...] → [0.65, 0.12, ...] 第 4 步: 加权求和所有的 V 输出_它 = 0.65×V_小猫 + 0.12×V_垫子 + ...
Attention 公式详解 组件
Attention(Q, K, V) = softmax( QKT / √dk ) V

第 1 步：QKT — Q 和 K 做矩阵乘法，得到"注意力分数矩阵"。Q 是"我在找什么"，K 是"我有什么"。两者越相似，分数越高。

第 2 步：÷ √dk — 缩放。为什么？如果向量维度大，点积结果也会大，softmax 会变得"太极端"——一个接近 1，其余全接近 0。除以 √dk 让数值更平滑。

假设 d_k = 64 原始分数: [128, 5, 3, 2] 除以 √64=8: [16, 0.625, 0.375, 0.25] softmax 后更平滑，不会过度集中

第 3 步：Softmax — 将分数归一化为概率分布（0-1，总和为 1）。

第 4 步：× V — 用注意力权重对 V 做加权求和，得到最终输出。

面试常考：不除以 √dk，softmax 输入值过大，梯度会趋近于 0，导致训练不稳定。这就是"Scaled Dot-Product Attention"中"Scaled"的由来。
多头注意力（Multi-Head Attention / GQA） 组件

一个注意力头只能从一个角度理解句子。就像看一幅画：

  • 头 1 关注语法关系："它"→"小猫"（代词指代）
  • 头 2 关注位置关系：相邻词之间的联系
  • 头 3 关注语义关系："舒服"→"垫子"

GQA（Grouped-Query Attention，分组查询注意力）：多个 Q 头共享同一组 K/V 头。

MiniMind 中：Q 有 8 个头，K/V 只有 4 个头，每 2 个 Q 头共享 1 个 K/V 头。

好处：减少 K/V 参数量和 KV Cache 显存占用，性能损失极小。是 MHA（全部独立）和 MQA（全部共享）的折中方案。

FFN（前馈神经网络） — SwiGLU 组件

Attention 是"收集信息"（开会讨论），FFN 是"消化信息"（会后各自回工位深入思考）。

MiniMind 使用 SwiGLU 激活函数：

FFN(x) = Wdown · ( SiLU(Wgate · x) ⊙ Wup · x )

通过一个"门控"机制让模型学会选择性传递信息，效果优于传统 ReLU。其中 ⊙ 表示逐元素乘法。

class FeedForward(nn.Module):
    def __init__(self, config):
        super().__init__()
        self.gate_proj = nn.Linear(config.hidden_size, config.intermediate_size, bias=False)
        self.down_proj = nn.Linear(config.intermediate_size, config.hidden_size, bias=False)
        self.up_proj = nn.Linear(config.hidden_size, config.intermediate_size, bias=False)

    def forward(self, x):
        return self.down_proj(F.silu(self.gate_proj(x)) * self.up_proj(x))
残差连接（Residual Connection） 组件
output = Layer(x) + x

把层的输入直接加到输出上。就像做菜时，即使加了很多调料，也保留食材的原味，不至于完全变味。

三大作用：

  1. 解决梯度消失 — 反向传播时，梯度可以通过"捷径"直接流回浅层
  2. 让深层网络可训练 — 没有残差连接，超过 20 层的网络几乎无法训练
  3. 保底不退步 — 最差情况，如果某层没学到有用的东西，输出至少等于输入
LayerNorm / RMSNorm（层归一化） 组件

对每个样本的特征做归一化，让均值为 0、方差为 1。就像考试后的"标准化评分"。

RMSNorm：LayerNorm 的简化版，只做缩放不减去均值：

RMSNorm(x) = x / √mean(x²) × γ

计算更快，效果相当。被 LLaMA、MiniMind 等广泛使用。

MiniMind 采用 Pre-Norm 位置（先归一化再做 Attention/FFN），而非原始 Transformer 的 Post-Norm。

Tokenizer（分词器） 组件

将原始文本转换为模型能处理的 token ID 序列。

例如： "今天天气" → token_ids: [234, 567, 567, 891]

🤗 Tokenizers 库支持 BPE、WordPiece、SentencePiece 等算法。分词是 NLP 流水线的第一步，直接影响模型对语言的理解粒度。

🚀 4. 进阶概念

pipeline() 函数 进阶

Hugging Face Transformers 库最简洁的 API。一行代码完成情感分析、文本生成、问答等任务：

from transformers import pipeline

# 情感分析
classifier = pipeline("sentiment-analysis")
result = classifier("I love this course!")
# [{'label': 'POSITIVE', 'score': 0.9999}]

# 文本生成
generator = pipeline("text-generation", model="gpt2")
generator("Once upon a time")

pipeline() 自动处理分词、模型推理、结果后处理的全流程。

Pre-Norm vs Post-Norm 进阶

📌 Post-Norm（原始 Transformer）

y = Norm( F(x) + x )

在子层之后归一化

需要 warm-up，训练不稳定

📌 Pre-Norm（现代标准）

y = F( Norm(x) ) + x

在子层之前归一化

训练稳定，无需 warm-up

MiniMind 使用 Pre-Norm + RMSNorm。Pre-Norm 已成为大模型标配，梯度流动更顺畅。

因果掩码（Causal Mask） 进阶

Decoder-Only 模型的关键机制。确保每个 token 只能关注当前位置及之前的 token，不能"偷看"未来的 token。

在训练时模拟推理时的生成场景。通常实现为一个上三角矩阵，未来位置被设为 -inf。

因果掩码矩阵（✓=可看，✗=不可看）: Token1 Token2 Token3 Token4 Token1 ✓ ✗ ✗ ✗ Token2 ✓ ✓ ✗ ✗ Token3 ✓ ✓ ✓ ✗ Token4 ✓ ✓ ✓ ✓
KV Cache 进阶

推理优化技术。在自回归生成中，每步只需计算新 token 的 Q，已生成 token 的 K 和 V 可以缓存复用，避免重复计算。

GQA 通过减少 K/V 头数，进一步降低 KV Cache 的显存占用。这是大模型推理加速的核心技术之一。

涌现能力（Emergent Abilities） 进阶

当模型规模超过某个阈值时，突然展现出小模型没有的能力：

  • 链式推理（Chain-of-Thought）
  • 代码理解与生成
  • 多语言翻译
  • 数学推理

这些能力不是被显式编程的，而是在大规模训练中"涌现"出来的，是 LLM 最令人惊讶的特性之一。

MiniMind 完整前向流程 进阶
输入文字: "今天天气" │ ▼ [Tokenizer] → token_ids: [234, 567, 567, 891] │ ▼ [Embedding] → 每个 token 变成 768 维向量 │ ▼ [RoPE 位置编码] → 给每个向量加上位置信息 │ ▼ [Block 1] → RMSNorm → GQA Attention → 残差 → RMSNorm → SwiGLU FFN → 残差 [Block 2] → 同上 [Block 3] → 同上 ... [Block 8] → 同上 │ ▼ [RMSNorm] → 最后做一次归一化 │ ▼ [lm_head] → 768 维 → 6400 维（vocab_size 每个 token 一个分数） │ ▼ [softmax + 采样] → 选出下一个 token

🎯 5. 面试考点

Q1: Attention 公式中为什么要除以 √dk？

当 Key 的维度 dk 较大时，Q·K 的点积结果方差约为 dk。数值过大会导致 softmax 的输入分布极端化——一个值接近 1，其余接近 0，梯度趋近于 0（梯度消失）。除以 √dk 将方差控制在 1 附近，保证梯度正常流动。这种做法称为 Scaled Dot-Product Attention。

Q2: 残差连接的作用是什么？如果去掉会怎样？

残差连接（y = F(x) + x）有三个作用：

  1. 缓解梯度消失 — 梯度通过"短路"直接传回浅层
  2. 使深层网络可训练 — ResNet 证明，无残差连接时深层网络性能反而下降（退化问题）
  3. 保证下限 — 即使某层完全没学到有用变换，输出至少等于输入

没有残差连接，Transformer 在 8 层以上时几乎无法训练。

Q3: Transformer 相比 RNN 的并行性优势体现在哪里？

RNN：必须先处理第 1 个 token 拿到隐藏状态，才能处理第 2 个。序列长度 N → N 步串行计算。

Transformer：Self-Attention 一次性计算所有 token 的关系（QKT 是一次矩阵乘法），所有位置并行处理。时间复杂度从 O(N) 步串行降为 O(1) 步并行。

代价：空间复杂度 O(N²)，因为需要存储 N×N 的注意力矩阵。

Q4: 什么是 GQA？MiniMind 为什么用它？

GQA（Grouped-Query Attention）让 K 和 V 的头数少于 Q 的头数——多个 Q 头共享同一组 K/V 头。

MHA = 每一个 Q 头有独立的 K/V 头（全独立，性能好但开销大）
MQA = 所有 Q 头共享 1 个 K/V 头（极省显存但性能有损失）
GQA = 折中方案：8 个 Q 头共享 4 个 K/V 头

好处：减少 K/V 参数量和 KV Cache 显存占用，性能损失极小。

Q5: Pre-Norm 和 Post-Norm 有什么区别？

Post-Norm（原始 Transformer）：y = Norm(F(x) + x) — 在子层之后做归一化

Pre-Norm（现代标准）：y = F(Norm(x)) + x — 在子层之前做归一化

Pre-Norm 优势：训练更稳定，不需要 warm-up，梯度流动更顺畅。理论上 Post-Norm 收敛后效果上限可能略高，但实践中 Pre-Norm 已成为大模型标配。
